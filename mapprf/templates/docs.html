Documentação

O Aplicativo MapPRF - Mapeamento de Ocorrências da Polícia Rodoviária Federal foi desenvolvido no âmbito do 2° Concurso para Dados Abertos do Ministério da Justiça & W3C Brasil. Este concurso aconteceu através de uma parceria estabelecida entre o Ministério da Justiça (Secretaria Executiva e Departamento da Polícia Rodoviária Federal), a Controladoria-Geral da União e o Ministério do Planejamento, Orçamento e Gestão (SLTI) tendo como referência a base de dados do Sistema de Informações Gerenciais (SIGER), mantida pelo Departamento de Polícia Rodoviária Federal.

Apesar da base de dados disponibilizada ser muito rica e possibilitar a realização de inúmeras análises, por questões de limitação de tempo, para este concurso, optamos por minimizar o escopo e trabalhar apenas com o conjunto de dados dos registros de acidentes nas rodovias federais. Além deste subconjunto de dados de acidentes de transito, foram utilizados dados da malha viária obtida a partir do Ministério dos Transportes e também dados oriundos do IBGE - Instituto Brasileiro de Geografia e Estatística.

Neste momento, no MapPRF estão disponíveis para análise os dados de acidentes registrados durante os anos de 2011, 2012 e primeiro semestre de 2013. Estes dados podem ser analisados através de mapas e gráficos interativos, conforme pode ser visualizado abaixo:

Figura 1: Página Inicial da Aplicação

Figura 2: Página de Acidentes por Município

Por questões de minimização de escopo, dificuldades no tratamento de inconsistências e na integração entre as diferentes bases de dados, para algumas análises, o MapPRF disponibiliza consultas mais detalhadas somente para a BR386, a qual está completamente contida no estado do Rio Grande do Sul. Entretanto, estas funcionalidades de análise podem ser aplicadas para as demais Rodovias Federais. Como exemplo, na imagem abaixo pode ser visualizada a BR386, com o indicador de acidentes por quilômetro, permitindo que o usuário faça a interação com elementos visuais para parametrização e atualização das informações.

Figura 3: BR386 – Dividida por trechos e com indicador de acidentes por Km

Por considerar que o georreferenciamento pode ser de grande importância para a análise de dados e também para implementação de funcionalidades futuras, na fase de integração de dados elaboramos um procedimento que utiliza interpolação para gerar uma coordenada geográfica (Latitude, Longitude) para identificar o local aproximado de um acidente. Este procedimento foi necessário porque pouquíssimos registros eram georreferenciados na base original. Como exemplo, na imagem abaixo (Fig5) podem ser visualizados o local aproximado de acidentes com mortes.

Figura 4: Local aproximado de todos os acidentes com mortes ocorridos na BR386

Além da análise de dados, o MapPRF permite que usuários com perfil de administrador possam realizar a manutenção e atualização de dados, conforme pode ser visualizado na imagem a seguir.

Tecnologias

Dentre as tecnologias utilizadas no desenvolvimento, podemos citar as seguintes tecnologias:

  • SGBD Postgresql
  • PostGis - Extensão Espacial para Postgresql
  • HTML
  • JavaScript
  • Django Framework
  • Bootstrap (Front-end Framework)
  • CanvasJS
  • CSS
  • Google Maps API
  • SASS
  • Bower
  • Grunt

O desenvolvimento deste aplicativo contou com o apoio da UFSM/CAFW - Universidade Federal de Santa Maria (Campus de Frederico Westphalen).
